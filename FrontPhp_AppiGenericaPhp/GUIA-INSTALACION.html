El navegador habla con el Frontend, el Frontend habla con la API, y la API habla con la base de datos.

Que es cada proyecto

ApiGenericaPhp Backend

Es la API REST (el backend). Es un servidor que recibe peticiones HTTP y responde con JSON. No tiene interfaz visual — no tiene botones, formularios ni tablas HTML. Solo recibe pedidos y devuelve datos.

Por ejemplo, si le pedis GET /api/producto, te responde un JSON con todos los productos de la base de datos. Si le mandas POST /api/producto con datos JSON, crea un producto nuevo.

Es generico: un solo controlador maneja TODAS las tablas. No necesitas crear codigo nuevo para cada tabla — cualquier tabla que exista en la BD se puede consultar con /api/{nombre_de_tabla}.

Se ejecuta con: C:\xampp\php\php.exe -S localhost:8000 -t public (servidor PHP integrado en el puerto 8000).

FrontPhp_AppiGenericaPhp Frontend

Es el frontend (la interfaz web). Es lo que el usuario ve en el navegador: formularios, tablas, botones, sidebar de navegacion. Esta hecho con PHP + HTML + Bootstrap (CSS).

El frontend no habla directamente con la base de datos. Cuando necesita datos (listar productos, crear una factura), le hace una peticion HTTP a la API usando cURL, recibe el JSON y lo convierte en HTML para mostrarlo en el navegador.

Lo sirve Apache (XAMPP) en el puerto 80. Se abre en: http://localhost/FrontPhp_AppiGenericaPhp/pages/home.php

Base de datos MariaDB

MariaDB es el motor de base de datos que viene con XAMPP (es compatible con MySQL). Guarda toda la informacion del sistema: productos, personas, clientes, vendedores, facturas, usuarios, roles, etc.

La base de datos se crea ejecutando un script SQL (bdfacturas_mariadb.sql) que crea 12 tablas, 5 triggers (acciones automaticas) y 5 stored procedures, con datos de ejemplo incluidos.

Se prende desde XAMPP → MySQL → Start (puerto 3306). Se administra visualmente en http://localhost/phpmyadmin.

Como se conectan entre si

1. El usuario abre en el navegador: localhost/FrontPhp_.../pages/producto.php
2. Apache ejecuta producto.php (frontend PHP)
3. producto.php usa cURL para pedir datos a la API:
    cURL GET → localhost:8000/api/producto
4. La API recibe la peticion en index.php
5. index.php la envia al controlador → servicio → repositorio
6. El repositorio ejecuta SQL con PDO:
    SELECT * FROM producto → MariaDB (puerto 3306)
7. MariaDB devuelve las filas → el repositorio las convierte a array PHP
8. La API responde JSON al frontend:
    {"tabla":"producto","total":8,"datos":[...]}
9. producto.php recibe el JSON, lo recorre y genera HTML con una tabla
10. Apache envia el HTML al navegador → el usuario ve la tabla de productos

Que es XAMPP y que hace cada boton

XAMPP es un programa que instala todo lo necesario para desarrollo web en Windows: Apache (servidor web), MariaDB (base de datos) y PHP (lenguaje). Se controla desde el XAMPP Control Panel:

Apache Servidor web. Sirve las paginas PHP del frontend al navegador. Puerto 80. Start para prenderlo.
MySQL Base de datos MariaDB (XAMPP lo llama "MySQL" pero es MariaDB). Puerto 3306. Start para prenderlo.
Admin Abre phpMyAdmin en el navegador (para ver y editar la base de datos visualmente).
Stop Apaga el servicio. Si dice Stop, significa que esta corriendo (verde).
PID / Port PID = numero de proceso (solo confirma que esta corriendo). Port = el puerto que usa.
Para este proyecto solo necesitas Apache y MySQL. FileZilla, Mercury y Tomcat no se usan — dejalos en Stop.

  Crear la base de datos con phpMyAdmin Solo la primera vez

Este paso solo se hace una vez. Si ya creaste la base de datos antes, saltalo.

0.1 — Prender MySQL y Apache desde XAMPP (ambos necesarios para phpMyAdmin):

Apache Running Puerto 80
MySQL Running Puerto 3306

0.2 — Abrir phpMyAdmin en el navegador:

http://localhost/phpmyadmin

Tambien se puede abrir haciendo clic en el boton Admin de la fila MySQL en XAMPP.

0.3 — Hacer clic en la pestana "Importar" (arriba en phpMyAdmin):

Bases de datos SQL Estado Importar Exportar
No seleccionar ninguna base de datos antes. El script ya crea la BD automaticamente.

0.4 — Hacer clic en "Seleccionar archivo" y buscar:

C:\xampp\htdocs\ApiGenericaPhp\script_bd\bdfacturas_mariadb.sql

0.5 — Verificar que el juego de caracteres diga utf-8 y hacer clic en el boton "Importar" (abajo).

Resultado: phpMyAdmin muestra un mensaje verde: "La importacion se ejecuto exitosamente". En el panel izquierdo aparece la base de datos bdfacturas_mariadb_local con 12 tablas.
Alternativa: crear la BD desde la terminal (sin phpMyAdmin)
> C:\xampp\mysql\bin\mysql.exe -u root --default-character-set=utf8mb4 -e "source C:/xampp/htdocs/ApiGenericaPhp/script_bd/bdfacturas_mariadb.sql"

Si no muestra ningun error, funciono. Silencio = exito.

Si necesitas resetear la BD: podes ejecutar la importacion de nuevo. El script borra todo y lo recrea desde cero con los datos de ejemplo. Ojo: los datos que hayas agregado se pierden.

Que hace el comando de la API

C:\xampp\php\php.exe -S localhost:8000 -t public
C:\xampp\php\php.exe El ejecutable de PHP. Se usa la ruta completa porque php solo no funciona si no esta en el PATH de Windows.
-S localhost:8000 Modo servidor. Levanta un servidor HTTP en localhost (esta PC), puerto 8000.
-t public Document root. La carpeta public/ es el punto de entrada. Ahi esta index.php que recibe todas las peticiones. El resto del codigo (src/, config/) queda protegido.

URLs utiles

localhost/FrontPhp_.../pages/home.php Pagina principal del frontend
localhost:8000/ Bienvenida de la API
localhost:8000/docs Swagger UI (probar la API visualmente)
localhost:8000/api/producto JSON con todos los productos
localhost/phpmyadmin Administrar la base de datos visualmente

Conceptos clave (breve)

XAMPP Paquete que instala Apache, MariaDB y PHP en Windows de una sola vez. Es como un "combo" para desarrollo web.
Apache Servidor web. Recibe peticiones del navegador y ejecuta archivos PHP. Sirve el frontend (las paginas con formularios y tablas).
MariaDB / MySQL Motor de base de datos. Guarda toda la informacion (productos, clientes, facturas) en tablas. MariaDB es compatible con MySQL.
API REST Un servidor que recibe peticiones HTTP (GET, POST, PUT, DELETE) y responde con JSON. Es el intermediario entre el frontend y la base de datos. El frontend nunca habla directo con la BD.
JSON Formato de texto para enviar datos: {"nombre":"Juan","edad":25}. La API responde en JSON y el frontend lo interpreta.
cURL Libreria de PHP para hacer peticiones HTTP. El frontend usa cURL para "hablar" con la API (como un navegador invisible).
PDO Libreria de PHP para conectarse a bases de datos. La API usa PDO para ejecutar SQL contra MariaDB de forma segura.
Trigger Codigo SQL que se ejecuta automaticamente cuando se inserta, actualiza o elimina una fila. Los triggers calculan subtotales y manejan el stock.
CRUD Las 4 operaciones basicas sobre datos: Create (crear), Read (leer), Update (actualizar), Delete (eliminar).
Puerto Numero que identifica un servicio en la PC. Es como el numero de puerta en un edificio: la BD esta en la puerta 3306, la API en la 8000, y Apache en la 80.

Solucion de problemas

La pagina no carga (error de conexion)

Apache no esta prendido. Ir a XAMPP → Apache → Start.

La pagina carga pero sin datos (tablas vacias)

La API no esta corriendo. Verificar que la terminal siga abierta con el servidor PHP. Si se cerro, volver al Paso 2.

La API responde pero sin datos (JSON vacio)

MySQL no esta prendido o la base de datos no existe. XAMPP → MySQL → Start. Si nunca creaste la BD, hacer el Paso 0.

"php" no se reconoce como comando

Usar la ruta completa: C:\xampp\php\php.exe en vez de solo php.

Puerto 8000 en uso (Address already in use)

Ya hay otra instancia corriendo. Cerrar esa terminal, o usar otro puerto: php -S localhost:8001 -t public (y cambiar el puerto en FrontPhp.../config.php).

Puerto 80 en uso (Apache no prende)

Otro programa ocupa el puerto 80 (Skype, IIS, otro servidor). Cerrar ese programa y reintentar.

Acentos se ven mal (caracteres raros)

Al crear la BD, agregar --default-character-set=utf8mb4 en el comando.
